# AI 书签管理器 - 产品设计文档

## 产品概述

**AI Bookmark Manager** 是一款智能书签管理浏览器插件，利用 AI 技术自动分类、整理和搜索用户的书签，解决书签杂乱无章、难以查找的痛点。

## 目标用户

- 📚 **重度浏览器用户** - 收藏了大量书签，难以管理
- 💼 **知识工作者** - 需要整理和查找大量参考资料
- 🎓 **学生和研究人员** - 需要分类管理学习资源
- 🚀 **效率追求者** - 希望提升书签管理效率

## 核心价值主张

1. **智能分类** - AI 自动理解书签内容并分类
2. **快速查找** - 语义搜索，理解用户意图
3. **简单易用** - 一键操作，无需手动整理
4. **隐私安全** - 数据本地存储，完全可控

## 功能设计

### 1. 书签同步与管理

#### 1.1 自动同步
- **功能**：实时同步浏览器原生书签
- **实现**：监听 Chrome Bookmarks API 事件
- **用户价值**：无需手动导入，自动保持最新

#### 1.2 书签 CRUD
- **功能**：完整的增删改查操作
- **界面**：
  - 侧边栏：快速浏览和访问
  - 管理页面：批量操作和详细编辑
- **用户价值**：灵活管理，适应不同场景

### 2. AI 智能分类

#### 2.1 自动分类
- **功能**：使用 OpenAI API 分析书签标题和 URL
- **分类维度**：
  - 主分类：技术、新闻、娱乐、购物、教育、工具等
  - 标签：多个细分标签（如：前端、React、教程）
  - 描述：一句话摘要
- **触发方式**：
  - 批量分类：一次性分类所有书签
  - 单个分类：右键菜单或编辑时
  - 自动分类：新增书签时（可选）
- **用户价值**：节省大量手动分类时间

#### 2.2 分类优化
- **智能合并**：相似分类自动合并
- **用户反馈**：支持手动调整分类
- **学习能力**：（未来）根据用户调整优化分类规则

### 3. 搜索与筛选

#### 3.1 普通搜索
- **功能**：基于标题、URL、标签、分类的模糊搜索
- **特点**：实时搜索，即时反馈
- **用户价值**：快速找到已知书签

#### 3.2 AI 语义搜索
- **功能**：理解用户意图，语义匹配
- **示例**：
  - 搜索"学习前端" → 找到所有前端相关教程
  - 搜索"买东西" → 找到所有购物网站
- **用户价值**：更自然的搜索方式

#### 3.3 高级筛选
- **维度**：
  - 按分类筛选
  - 按标签筛选
  - 按时间筛选
  - 组合筛选
- **用户价值**：精确定位目标书签

### 4. 用户界面

#### 4.1 侧边栏
- **布局**：
  - 顶部：搜索框 + 操作按钮
  - 中部：树形书签列表
  - 底部：统计信息
- **交互**：
  - 点击打开书签
  - 右键菜单：编辑、删除、分类
  - 拖拽排序（未来）
- **设计原则**：简洁、高效、信息密度适中

#### 4.2 管理页面
- **布局**：
  - 左侧：工具栏和统计
  - 右侧：书签列表（列表/网格视图）
- **功能**：
  - 批量选择和操作
  - 查找重复书签
  - 导入导出
- **设计原则**：功能完整、操作直观

#### 4.3 配置页面
- **内容**：
  - OpenAI API 配置
  - 功能开关
  - 主题设置
  - 数据管理
  - 统计信息
- **设计原则**：清晰分组、易于理解

### 5. 快捷操作

#### 5.1 快捷键
- `Cmd/Ctrl + Shift + S`：保存当前页面
- `Cmd/Ctrl + Shift + B`：打开侧边栏

#### 5.2 右键菜单
- 在书签上右键：打开、编辑、分类、删除

#### 5.3 一键操作
- 保存当前页面
- AI 批量分类
- 查找重复

### 6. 数据管理

#### 6.1 本地存储
- **存储内容**：
  - 书签元数据（分类、标签、描述）
  - 用户配置
  - OpenAI API Key（加密）
- **存储位置**：Chrome Storage API
- **优势**：隐私安全、响应快速

#### 6.2 导入导出
- **导出格式**：
  - JSON：包含所有元数据
  - HTML：标准书签格式
- **导入格式**：
  - JSON：恢复元数据
- **用户价值**：数据备份、跨设备迁移

#### 6.3 数据清理
- 清除分类数据（保留书签）
- 查找并删除重复书签
- 检测失效链接（未来）

## 技术架构

### 前端技术栈
- **框架**：Vanilla JavaScript（轻量、快速）
- **样式**：CSS3 + CSS Variables（主题切换）
- **API**：Chrome Extension APIs

### 后端服务
- **AI 服务**：OpenAI API
- **模型选择**：
  - 推荐：GPT-4o Mini（性价比高）
  - 备选：GPT-3.5 Turbo、GPT-4o

### 数据流
```
用户操作 → UI 组件 → 业务逻辑 → Chrome APIs / OpenAI API
                                ↓
                          Storage Manager
                                ↓
                          Chrome Storage
```

## 竞品分析

### 现有解决方案
1. **浏览器原生书签**
   - 优势：简单、免费
   - 劣势：功能有限、无智能分类

2. **Pocket / Instapaper**
   - 优势：稍后阅读、跨平台
   - 劣势：需要单独保存、非书签管理

3. **Raindrop.io**
   - 优势：功能丰富、美观
   - 劣势：需要账号、部分功能收费

### 我们的优势
- ✅ AI 智能分类
- ✅ 完全本地化
- ✅ 隐私优先
- ✅ 开源免费
- ✅ 无需账号
